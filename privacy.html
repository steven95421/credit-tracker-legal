Privacy Policy

Effective date: May 17, 2026 · Last updated: May 17, 2026

1. Overview

Credit Tracker (the "Application") is a self-hosted, open-source personal finance tool that helps a single user track unused statement credits on their own credit cards. The Application is operated and used exclusively by Chiyu Yang ("the Operator") for personal financial management. There are no other users, customers, or third-party clients.

This privacy policy explains what data the Application accesses, how it is stored, and how it is used. It is published to meet the requirements of third-party data providers (such as Plaid Inc.) whose APIs the Application integrates with.

2. Data we collect

The Application accesses the following categories of data, exclusively about the Operator's own accounts:

  • Financial account information obtained via the Plaid API: institution name, account name, account type/subtype, account mask (last 4 digits), and Plaid account identifiers.
  • Credit and debit card transactions: transaction ID, date, amount, currency, merchant name, transaction name, pending status, and Plaid's transaction categorization.
  • User-supplied metadata: manual mappings between Plaid accounts and statement-credit catalog entries, and manual overrides indicating that a credit has been used or reset.

The Application does not access or store:

  • Bank usernames or passwords (handled exclusively inside the Plaid Link UI, never seen by the Application).
  • Full account numbers or full card numbers (only the masked last 4 digits returned by Plaid).
  • Social Security numbers, identity-verification data, or biometric data.
  • Location, contact lists, device sensors, or other operating-system data.

3. How data is used

Data is used solely to:

  • Display the Operator's transactions and account balances within the locally-running web dashboard.
  • Match transactions against a catalog of known statement credits to compute remaining benefits per period.
  • Send reminders to the Operator about credits that are about to expire.

Data is not sold, rented, licensed, shared with advertisers, or used to train any AI model.

4. Data storage and security

  • All data is stored locally on the Operator's personal computer in a SQLite database file.
  • Plaid access tokens are encrypted at the application layer using Fernet (AES-128-CBC + HMAC-SHA256). The symmetric key is stored in a separate file with owner-only permissions (chmod 600) and is excluded from version control.
  • The host device additionally runs macOS with FileVault full-disk encryption enabled, providing a second layer of encryption at rest.
  • The host device requires Touch ID / password authentication to unlock.
  • All HTTP API calls to the locally-running dashboard require a server-generated bearer session token. CORS is restricted to localhost origins only.
  • Communication between the Application and the Plaid API uses TLS 1.2 or higher, as enforced by the Plaid Python SDK.
  • The .env file containing Plaid client credentials is excluded from version control.
  • No data is transmitted to any server other than Plaid's API endpoints.

5. Third-party services

The Application uses the Plaid API to securely connect to the Operator's financial institutions. Plaid's privacy practices are governed by Plaid's own End User Privacy Policy and are independent of this policy. By using the Application, the Operator authorizes Plaid to access account data on the Operator's behalf.

6. Data retention and deletion

The Operator may delete all stored data at any time by:

  • Deleting the SQLite database file (data/tracker.db) to remove transactions and overrides.
  • Calling the Plaid /item/remove endpoint, or revoking access from within each bank's online portal, to revoke the Application's access to financial institutions.
  • Deleting the Application directory entirely.

No automated retention is enforced; the Operator chooses when to delete data. Because the Application has only one user (the Operator), there are no third-party data subject rights to administer.

7. Consent

Because the Operator is the only user, the Operator's act of running and configuring the Application constitutes informed consent to the collection and storage of their own financial data. The Application will never share data on behalf of any other user.
